# 旧平台 vs ciciERP 界面与数据展示对比

> 文档日期：2026-03-31  
> 分析依据：旧平台截图 7 张（docs/code_from_other_platform/）+ ciciERP 现有代码

---

## 一、旧平台（Mini ERP 运营中心）功能全貌

> **重要说明**：`app.py` 仅是旧平台的「商品价格表」一个 Tab，旧平台实际是一个更完整的多 Tab 系统。

### 1.1 导航结构

旧平台顶部导航包含 6 个模块 Tab：

| Tab | 功能描述 |
|-----|---------|
| 商品价格表 | 产品价格查看（app.py 对应页面） |
| 库存 | 库存管理 |
| **销售** | 销售统计 + 速卖通/阿里订单（含利润分析）|
| **客户信息** | CRM + 询盘管理 + AI录入 |
| 采购 | 采购管理 |
| 发票 | 发票管理 |

### 图2：速卖通订单明细（RMB Tab）

**订单表格列（订单行级别，每行 = 一个产品）：**

| 列 | 说明 |
|----|------|

## 四、ciciERP 相对旧平台的优势

| 优势点 | 说明 |
|--------|------|
| 完整 CRUD | 产品/供应商/客户/订单均可增删改，旧平台基本只读 |
| 采购管理 | 完整采购单流程，旧平台仅有 Tab 入口 |
| 物流管理 | 物流公司 + 发货单 + 运踪，旧平台无对应 |
| PI/CI 发票 | 完整发票管理模块，旧平台仅有发票 Tab |
| 用户权限 | JWT + RBAC，旧平台无权限体系 |
| 对接 API | cicishop 等外部平台接口，旧平台无 |
| 数据持久化 | SQLite 数据库，旧平台依赖 Excel 文件 |

## 五、建议优先补齐的功能（按价值排序）

### 🔴 高优先级（直接影响日常使用）

| 功能 | 所属模块 | 实现难度 | 说明 |
|------|---------|---------|------|
| 订单列表增加利润/成本列 | 订单 | 低 | order_items 表已有 cost_price 字段 |
| 订单亏损标记 | 订单 | 低 | 毛利 < 0 时标红/标记 |
| 来源平台显示为文字 | 订单 | 低 | platform 字段已有，改显示即可 |
| 数据分析：双币种销售汇总 | 分析 | 中 | CURRENCY_AND_ANALYTICS_PLAN.md 已规划 |
| 月度利润/毛利率图表 | 分析 | 中 | 同上，已有规划文档 |

### 🟡 中优先级（提升运营效率）

| 功能 | 所属模块 | 实现难度 | 说明 |
|------|---------|---------|------|
| 客户跟进状态 + 跟进日期 | CRM | 中 | 需在 customers 表增加字段 |
| 今日跟进/逾期提醒卡片 | CRM | 中 | 依赖跟进日期字段 |
| 客户关联询盘产品 | CRM | 中 | 需新增 inquiries 表或扩展 customers 表 |
| 产品列表统计卡片（4个指标）| 产品 | 低 | 前端聚合计算即可 |
| 产品价格区间筛选 | 产品 | 低 | 前端 + 查询参数 |

### 🟢 低优先级（锦上添花）

| 功能 | 所属模块 | 实现难度 | 说明 |
|------|---------|---------|------|
| AI 快速录入客户（询盘解析）| CRM | 高 | 需接入 Claude API |
| 订单整体导出 Excel | 订单 | 中 | 类似现有 PI/CI 导出 |
| 库存不足订单自动备注 | 订单 | 中 | 创建订单时检查库存 |
| 客户优先级分级 | CRM | 低 | 增加 priority 字段 |
