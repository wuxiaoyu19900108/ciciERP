## 四、产品价格优化审核

### 4.1 审核结果明细

| 需求项 | 状态 | 说明 |
|--------|------|------|
| 价格输入框显示参考价格提示 | ⚠️ 部分通过 | 有标签但未实现动态 placeholder |
| 查看历史价格按钮 | ✅ 通过 | 有按钮和弹窗显示 |
| 历史价格从已完成订单获取 | ⚠️ 部分通过 | SQL 排除了取消订单，但未限制为已完成订单 |

### 4.2 代码位置

- **历史价格 API**: `crates/api/src/routes/products.rs:202-227`
- **SQL 查询**: `crates/db/src/queries/orders.rs:584-608`
- **前端 JavaScript**: `web.rs:2967-2996`

### 4.3 问题详情

#### 问题 4: 参考价格未实现动态 placeholder (低优先级)

**需求原文**:
> 选择产品后，价格输入框显示 placeholder 为参考价格

**当前实现**:
```html
<label>单价(USD) <span id="priceTip">(参考价格)</span></label>
```

**问题**:
- 标签显示静态文本 "(参考价格)"
- 没有从 `product_prices` 表获取实际参考价格
- 没有动态设置 placeholder

**建议修复**:
1. 产品选择时调用 `/api/v1/products/:id/prices/summary` 获取参考价格
2. 设置 `placeholder="参考价格: $XX.XX"` 或 `placeholder="请输入报价"`

#### 问题 5: 历史价格数据源不准确 (中优先级)

**需求原文**:
> WHERE o.status IN (3, 4, 5) -- 已付款、已发货、已收货

**当前实现** (`orders.rs:585-600`):
```sql
WHERE oi.product_id = ? AND o.order_status != 6
```

**问题**:
- 当前查询只排除了取消订单 (status != 6)
- 按需求应该只包含已完成订单 (status IN (3, 4, 5))
- 未成交订单的价格也会被当作历史成交价格

**建议修复**:
```sql
WHERE oi.product_id = ? AND o.order_status IN (3, 4, 5)
```
